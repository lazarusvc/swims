# Running a Report

## Steps

1. Navigate to **Reports** in the sidebar.
2. Click the report you want to run.
3. If the report has **parameters**, a form will appear at the top of the report page. Fill in any required parameters (marked with `*`) and any optional ones you need.
4. Click **Run Report** (or **View**).
5. The report loads in the viewer panel below the parameter form.

## Common Parameters

Most reports will ask for some or all of the following:

| Parameter | Description |
|-----------|-------------|
| **Date From / Date To** | The date range to include in the report |
| **Programme** | Filter results to a specific social welfare programme |
| **Status** | Filter by case or application status |
| **Officer / User** | Filter by the assigned case worker or staff member |

## Viewing the Report

Reports display inline in SWIMS — no separate browser window or download is needed to view the HTML version. Scroll within the report viewer to read the full report.

### Navigating Multi-Page Reports

For large reports, page navigation controls appear at the top or bottom of the report viewer. Use these to move between pages.

### Exporting to PDF

Click the **Download PDF** button (where available) to download the report as a PDF file for printing or sharing. The PDF opens or downloads based on your browser settings.

## Report Takes a Long Time to Load

Complex reports querying large datasets may take 30–60 seconds on first load. This is normal. Do not refresh the page while the report is loading — wait for it to complete.

If a report consistently times out, notify your administrator.

## Report Not Showing Correct Data

- Check your parameter values — particularly date ranges. An incorrect **Date From / Date To** is the most common cause of unexpected results.
- Ensure you selected the correct programme or status filter.
- If you believe the underlying data is wrong, raise it with your administrator or the relevant case worker.
