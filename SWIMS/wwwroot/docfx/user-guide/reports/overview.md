# Reports — Overview

The Reports module gives authorised staff access to operational reports generated by **SQL Server Reporting Services (SSRS)**. Reports are displayed directly within SWIMS without needing to access the report server separately.

## Accessing Reports

Navigate to **Reports** in the sidebar (requires `Reports.View` permission).

You will see a list of all available reports, organised by name. Click any report to open it.

## Report List

Each report in the list shows:

- **Report name**
- **Description** — a brief explanation of what the report shows
- A **Run** or **View** button

## Running a Report

See [Running a Report](running.md) for step-by-step instructions.

## Who Can Access Reports

| Permission | Access |
|-----------|--------|
| `Reports.View` | Can view and run all active reports |
| `Reports.Admin` | Can manage report definitions and parameters (Admin area) |

Contact your administrator if you cannot see the Reports section or a specific report you need.

## Report Administration

Administrators can manage the report catalogue via **Admin → Reports**:

- **Add** new SSRS reports to the catalogue
- **Edit** report names, descriptions, and parameters
- **Activate / Deactivate** reports (deactivated reports are hidden from users)

See [Administration → System Settings](../admin/system-settings.md) for configuration details.
